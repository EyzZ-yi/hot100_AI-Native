# ADR-001: 分阶段资源调度策略

日期：2026-06-01
状态：草案（基于个人 dogfooding 假设，待数据验证）

## 背景
用户画像显示两个核心痛点：
1. 看完题解觉得懂了，自己写不出来
2. 不知道下一步该做什么

## 决策
不做"超越代随的题解生成器"，而做"分阶段资源调度 agent"：

- 首次学新算法 → 代随静态资源（图解 + 文章 + 视频）
- 复习卡住 → LLM 基于用户当前思路的针对性 hint
- 已掌握 → 推荐进阶题

## 理由
基于个人刷代随时的实践观察：
- 首次接触时，图解的认知效率 > LLM 文字解释
- 复习卡住时，需要"针对当前错误的点拨"，静态资源做不到

## 待验证
当前判断基于 ~5 次个人体验。需要 30+ 数据点的 evaluation。

## 评估方法
每道题刷完记录 2 个字段：
- 首次时哪种资源更有效
- 复习卡住时哪种资源更有效

## 后果
-  不和代随竞争题解质量（不可能赢）
-  简化 LLM 部分的工程难度
-  增加对代随的依赖（注意版权和链接稳定性）
