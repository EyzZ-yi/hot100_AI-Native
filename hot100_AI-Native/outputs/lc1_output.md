# LC1 两数之和

好的，我们开始讲解这道非常经典的 LeetCode 第一题：**两数之和**。

---

### 1. 思路

这道题之所以用**哈希表（HashMap）**，是因为我们需要快速判断 `target - nums[i]` 这个数字是否已经出现在数组里，并且能瞬间拿到它的下标。如果暴力两层循环，虽然也可以，但时间复杂度是 O(n²)，对于大数据量不够高效。哈希表能将查找时间降到 O(1)，所以我们遍历一次数组，边存边查，就能在 O(n) 时间内解决。

### 2. Java 代码（带关键步骤注释）

```java
import java.util.HashMap;
import java.util.Map;

public class Solution {
    public int[] twoSum(int[] nums, int target) {
        // 创建哈希表：key 存储数组元素值，value 存储对应的下标
        Map<Integer, Integer> map = new HashMap<>();

        // 遍历数组
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];  // 需要寻找的另一个数

            // 检查哈希表中是否已经存在这个互补数
            if (map.containsKey(complement)) {
                // 找到答案，返回 [互补数的下标, 当前下标]
                return new int[]{map.get(complement), i};
            }

            // 如果没找到，将当前元素和它的下标存入哈希表
            map.put(nums[i], i);
        }

        // 根据题意，一定存在一个答案，所以不会执行到这里
        // 但为了编译通过，返回一个空数组
        return new int[0];
    }
}
```

---

### 3. 时间复杂度 + 空间复杂度

- **时间复杂度：O(n)**  
  我们只遍历了一次数组，每次在哈希表中进行 O(1) 的查找和插入操作，n 是数组的长度。

- **空间复杂度：O(n)**  
  最坏情况下，我们需要把数组里几乎所有元素存进哈希表，额外空间取决于哈希表的大小。

---

### 4. 关键点

- **哈希表的使用**：利用哈希表 O(1) 的查找能力来替代暴力循环。
- **边遍历边存储**：我们不需要先存完所有元素再查，而是在遍历过程中“边走边查、边走边存”，既节省时间，也能避免同一个元素被重复使用。
- **查找互补数**：核心思想是 `target - nums[i]` 是否已经出现过，这比两数相乘、相除等其他运算更直观且不易出错。
