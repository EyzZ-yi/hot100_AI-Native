# LC202 快乐数

好的，我们开始讲解这道题。

## 1. 思路

判断一个数是否是快乐数的核心是 **检测是否会进入循环**（而非到达1）。因为一个数通过“各位平方和”计算后，要么最终会变成 1，要么会在若干次的迭代中进入一个不包含1的循环。所以**计算下一个数字**和**哈希集合记录出现过的数字**是主要手段。  
如果没有使用集合记录历史，也可用 **快慢指针（Floyd判圈法）** 探测循环，但这里使用哈希集合更直接易懂。

参考 [代码随想录 · 快乐数题解](https://programmercarl.com/0202.%E5%BF%AB%E4%B9%90%E6%95%B0.html)

## 2. Java 代码

```java
import java.util.HashSet;
import java.util.Set;

public class Solution {
    public boolean isHappy(int n) {
        Set<Integer> seen = new HashSet<>();  // 记录已经出现的数字，检测循环
        
        while (n != 1 && !seen.contains(n)) {
            seen.add(n);                // 加入当前数
            n = getNext(n);             // 计算下一个数（各位平方和）
        }
        
        return n == 1;                  // 循环结束可能因n=1（快乐数），或发现循环（非快乐数）
    }
    
    // 辅助函数：计算一个整数的各位数字平方和
    private int getNext(int num) {
        int sum = 0;
        while (num > 0) {
            int digit = num % 10;       // 取出个位
            sum += digit * digit;       // 平方后累加
            num /= 10;                  // 去掉个位
        }
        return sum;
    }
}
```

## 3. 时间复杂度 + 空间复杂度

- **时间复杂度**：O(log n)  
  每次 getNext 操作的时间与十进制位数成正比，接近 O(log n)。循环次数的上限取决于数字变化范围，但实验与分析表明对所有输入数字会快速收敛到较小范围，可以看作常数或对数级别。整体视为 O(log n)。

- **空间复杂度**：O(log n)  
  哈希集合所需存储已经出现过的数字，最多存储一个循环内的所有不同数字，数量不大，与 n 的位数有关（实际规模很小），分析为 O(log n)。

## 4. 关键点

- 理解无限循环等价于有环，使用哈希集合检测环是否存在。
- 学会 **数字拆位 + 平方和** 的技术（getNext 函数）。
- 本题考察 **问题转化**：将抽象“无限循环”等价为“重复值出现”。
- 熟悉集合用于去重/判重的思想。
