# LC206 反转链表

好的，同学们，我们来看这道非常经典的反转链表题。这是链表操作的基础，几乎所有链表面试都会涉及类似的变形。

## 1. 思路

这道题的核心思想是**改变每个节点的指针方向**，让原本指向下一个节点的指针指向前一个节点。因为链表无法像数组那样直接反向访问，所以我们需要用**迭代法**或**递归法**来挨个修改指向。

- 迭代法（推荐）：使用三个指针（`prev`、`cur`、`next`），每次让当前节点指向前一个节点，然后依次向前移动。简单直观。
- 递归法：递归到链表末尾，然后从后往前修改指向，稍微难理解一些但对于理解递归很有帮助。

这道题的思路与[代码随想录 - 206.反转链表](https://programmercarl.com/0206.%E7%BF%BB%E8%BD%AC%E9%93%BE%E8%A1%A8.html)一致，推荐大家去看看。

## 2. Java代码（迭代法）

```java
public ListNode reverseList(ListNode head) {
    ListNode prev = null;  // 前一个节点，初始为 null（新的末尾指向 null）
    ListNode cur = head;   // 当前要处理的节点

    while (cur != null) {
        ListNode next = cur.next; // 暂存下一个节点，防止链表断开后丢失
        cur.next = prev;          // 核心步骤：当前节点指向前一个节点，反转方向
        prev = cur;               // prev 前移一位
        cur = next;               // cur 前移一位
    }
    // 循环结束后，prev 指向原链表的最后一个节点，也就是新链表的头节点
    return prev;
}
```

递归法（可选理解）：

```java
public ListNode reverseList(ListNode head) {
    if (head == null || head.next == null) {
        return head; // 递归基：只剩一个节点时直接返回
    }
    ListNode newHead = reverseList(head.next); // 递归反转后续链表
    head.next.next = head; // 让后一个节点指回当前节点
    head.next = null;      // 当前节点不再指向后面（避免循环）
    return newHead;
}
```

## 3. 时间复杂度 + 空间复杂度

| 方法 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 迭代法 | O(n) | O(1) |
| 递归法 | O(n) | O(n)（递归调用栈深度） |

- 时间复杂度 `O(n)`：无论哪种方法，都需要遍历所有 n 个节点一次。
- 空间复杂度：迭代法仅用几个指针，常数空间；递归法因为要保存 n 层递归调用信息，需要 O(n) 的栈空间。

## 4. 关键点

这道题主要考察以下几个知识点：

1. **链表的指针操作**：需要非常清晰地理解每个节点只保存一个 `next` 指针，反转就是修改这个指针的指向。
2. **防止链表断裂**：在修改 `cur.next` 之前，必须用临时变量保存下一个节点位置。
3. **边界条件处理**：空链表（`head == null`）和单节点链表的情况，反转结果仍是自身。
4. **迭代 vs 递归的编程思维**：理解这两种方式如何解决同一问题，特别要能用手画图走一遍过程。

练习建议：先在纸上画出链表，用箭头演示每一步指针变化，代码自然就容易写对了。
