# LC242 有效的字母异位词

好的，我们一起来分析这道经典的算法题。

## 1. 思路

判断字母异位词的核心在于：两个字符串中每个字符出现的次数必须完全相同。因为题目中只包含小写字母（如果没明确说明，通常代码也要考虑扩展性），所以我们使用一个长度为 26 的数组作为计数器，先遍历第一个字符串统计每个字母出现次数，再遍历第二个字符串逐一减少次数，最后检查数组是否全部归零即可。这样只用到常量数组，不依赖哈希表的动态扩容，性能更好。

## 2. Java 代码（带关键注释）

```java
public boolean isAnagram(String s, String t) {
    // 长度不同，一定不是异位词
    if (s.length() != t.length()) {
        return false;
    }

    // 用一个长度为26的数组记录每个字母出现的次数（假设全是小写字母）
    int[] record = new int[26];

    // 遍历第一个字符串，字符出现次数+1
    for (int i = 0; i < s.length(); i++) {
        record[s.charAt(i) - 'a']++;
    }

    // 遍历第二个字符串，字符出现次数-1
    for (int i = 0; i < t.length(); i++) {
        record[t.charAt(i) - 'a']--;
    }

    // 检查所有位置是否都归零
    for (int count : record) {
        if (count != 0) {
            return false;
        }
    }

    return true;
}
```

## 3. 时间复杂度 + 空间复杂度

- **时间复杂度：O(n)**  
  其中 n 是字符串的长度，两个独立的单循环（实际上合并就是遍历一次 s 一次 t），因此是 O(n)。

- **空间复杂度：O(1)**  
  额外只用了固定大小（26）的数组，不随输入规模变化，因此是常量空间。

## 4. 关键点（考察知识点）

- **哈希思想的应用**：用数组模拟哈希表，尤其适用于数据范围有限（小写字母）的情况。
- **计数统计与相互抵消**：不是单纯计数，而是用“一加一减”的方式快速判断两次计数是否完全匹配。
- **长度前置判断**：长度不同则直接返回 false，避免无谓计算。
- **数组索引映射**：字符转索引（`char - 'a'`）是处理字母统计的常用技巧。
