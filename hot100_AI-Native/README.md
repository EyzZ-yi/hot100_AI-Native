# Hot100 学习伙伴项目 - 用户画像 v0.1

## Part 1：我自己刷 hot100 时的真实痛点

- 痛点 1：我做完一道双指针题后，不知道下一道该做什么——回代码随想录翻目录要 10 分钟，回 LeetCode 看推荐题不一定相关
- 痛点 2：我看完题解觉得"懂了"，但隔一周自己写还是写不出来
- 痛点 3：有些时候刷完几道题，不知道干什么了，没有方向
- 痛点 4: hot100分类一般，不够细致，有些前置知识根本不一样，卡住纯粹浪费时间（比如你没写过dp根本不会写）

## Part 2：代码随想录在这些场景下为什么帮不到我

- 1：他是静态的，需要自主学习能力和规划能力
- 2：单向输出，没有"测验你"的机制
- 3: 没有用户状态，不记得你之前犯过的错

## Part 3：如果一个 AI 工具能帮我，它的核心交互是什么样

例子：
- 交互 1：
        我刚做完一道题（ac），用户粘贴代码
              ↓
        提醒用户给出题号
              ↓  
          鼓励一下user
              ↓
        给出推荐的题号和难度，提醒用户输入选择
              ↓
            用户给出题号
              ↓
        判断用户是否有前置知识
              ↓
        NO补充前置知识或者给出链接，YES结束
- 交互 2：
        发给我不会的题目，给出难度，我先给出思路和伪代码，还有代码随想录的视频链接和网页链接
                                 ↓
                    判断是否做过类似的题目和前置知识
                ↓ yes                      ↓no
            给出之前的题号和伪代码                 给出伪代码代码随想录的链接

- 交互 3：
                    用户提交错误代码
                        ↓
Level 1：你的思路是对的，但有个细节问题，再看看 X 行（不直接说错在哪）
                        ↓ 用户还卡住
Level 2：提示你忽略了某种边界情况（具体说是哪种，但不给修复方案）
                        ↓ 用户还卡住
Level 3：给出修复方向（"你应该用 X 而不是 Y"，但不给完整代码）
                        ↓ 用户还卡住
Level 4：给出完整 AC 代码 + 详细讲解为什么之前的写法不对
- 交互 4：
                识别用户在做“记忆里的错题”
                ↓  ac               ↓wa
            错题根据规则降低权重        提交错误代码的流程，增加权重
                

## Part 4：什么情况下我会放弃这个工具

- 1:自己找的题解或者伪代码太垃圾了
- 2:推荐的题目关联度不高
- 3:自定义api，好的api太贵，差的api不好用
- 4:用户用了 1 周后失去新鲜感不再打开

## Part 4:优势或者功能

- 1：我可以给出用到一样算法和思路的题目
- 2：记录看过题解的题目，或者时间过长的题目，隔一段时间（艾宾浩斯遗忘曲线？）推送给user，让他先讲为代码，为user量身定做专属ai
- 3: agent推荐路线感觉可以参考代码随想录的学习顺序
- 4：user第一次使用的时候可以描述一下自己的学习阶段（刚学完数据结构，刷了几道leetcode（给出题号））初始化“记忆”
- 5：用户无法ac的题目，分析错误原因，比如不清楚“java是值传递”导致的，记录在错误“记忆里”下次复习给出，如果只是拼写或语法错误，则不记录
- 6: 每道题标注有考过这道题的大厂

## Day 1 Reflection（2026-05-31）

### 技术认知
- RAG 不是项目核心，是 20% 的辅助组件
- 项目核心数据流：70% SQLite 状态管理 + 20% LLM 生成 + 10% RAG 检索

### 项目护城河识别  
- 不在技术栈（公开）
- 在向量库的内容（独家资产）：
  - 错误模式库（50-100 条）
  - 题目关系图（100-200 条边）
  - 概念解释库

## RAG用处
用户提交错误代码 →
LLM 分析后判断：错误根因是"对 Java 值传递的误解" →
RAG: 用"Java 值传递"语义检索 →
返回向量库里的【常见错误模式描述】+【相关例题】 →
LLM 综合生成针对这个用户的辅导
