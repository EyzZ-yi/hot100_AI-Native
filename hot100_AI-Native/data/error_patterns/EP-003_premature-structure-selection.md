# EP-003: 先选熟悉的数据结构，再套问题

## 触发场景
分析阶段，刚看到题目就想到自己近期刚学的结构，直接开始写。

## 错误表现
- 三数之和用 hash 做去重，导致去重逻辑极其复杂（正确做法：排序 + 双指针）（LC15）
- 长度最小的子数组用快慢指针，窗口无法正确收缩（正确做法：滑动窗口）（LC209）
- 前 K 高频元素花了 45min 才识别出「堆」，早期尝试了排序等方案（LC347）

## 根因
思路顺序是"我会 X → 能不能套 X？"，而不是"这道题的输入/输出特征是什么 → 对应哪种结构？"

## 教学策略
1. 分析流程：先确认输入/输出/约束 → 再识别"特征信号"（有序?去重?前k?嵌套配对?）→ 最后选结构
2. 特征信号速查：
   - 有序数组 + 求和/差 → 双指针
   - 连续子数组 + 条件收缩 → 滑动窗口
   - 前 K / TopK → 堆
   - 去重组合 → 排序 + 双指针（不是 hash）
   - 嵌套/配对/最近优先 → 栈
3. 如果强行套熟悉结构后代码变复杂，是放弃的信号，不是"再调一下"的信号
