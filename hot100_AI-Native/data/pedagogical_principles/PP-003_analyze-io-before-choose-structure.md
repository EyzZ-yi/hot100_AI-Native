# PP-003: 理解输入/输出后再选数据结构

## 触发场景
拿到题目后直觉性地选择了刚刚学过的结构，写到一半发现代码越来越复杂。

## 原则内容
分析流程：**输入是什么 → 输出是什么 → 约束是什么 → 特征信号 → 选结构**。  
不允许反向操作：先选结构再硬套问题。

## 底层理由
算法的本质是数据结构的选择。结构选对，代码自然简洁；结构选错，写出的每一行都是在对抗结构本身的局限。代码复杂度上升是错误选择的信号，不是"继续优化"的信号。

## 特征信号速查
| 特征 | 首选结构 |
|------|---------|
| 有序数组 + 去重组合 | 排序 + 双指针 |
| 连续子数组 + 条件收缩 | 滑动窗口 |
| 前 K / TopK | 堆 |
| 嵌套 / 配对 / 最近优先 | 栈 |
| 先进先出 / 层序 | 队列 |
| O(1) 存在查询 | 哈希 |

## 应用方式
1. 读完题后先复述"输入/输出/约束"，再找特征信号
2. 如果代码超过 30 行还没方向，停下来重新分析输入输出
3. 看到代码越来越复杂是放弃当前结构选择的信号
