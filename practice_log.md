## LC 84 柱状图中最大的矩形


### 首次（2026-06-08）
- 自评掌握度：1/5
- 用时：50min
- 状态：看题解
- 思路：
- 卡点
1. 高度取错:拿成了「弹栈后的新栈顶」的高度——今天犯了两次。
   边界柱比 mid 矮,正因如此才当边界,它根本不在矩形里。
2. 宽度取错两处:① 右边界误用 mid(该用 i);② 把左边界 L 含了进去(+1,该是 -1 开区间)。
3. 最初代码用 Arrays.sort 囤候选再取最大 —— 暴露了不相信「每根柱子结算一次就够」,所以手动兜底。
4. 纠结栈是「递增还是递减」—— 伪卡点。名字随约定翻转,抓它就是给自己挖坑。

- 关键学到的
1. 结构(只记这个,不记名字):底部小、顶部大;遇到比栈顶矮的就弹栈。
2. 弹栈瞬间,左右边界同时锁定:右 = 当前下标 i(右边第一根更矮),左 = 弹栈后的新栈顶 L(左边第一根更矮)。
3. 公式:area = h[mid] × (i − L − 1)。高度只来自 mid 自己,宽度两端都开。
4. 右边界为什么是 i 不是 mid:mid 能在栈里活到 i 才被弹,说明 mid..i−1 全部 ≥ h[mid],都属于这块矩形,所以能铺到 i−1。
5. 完备性(为什么取 max 不漏):任取一个矩形 R,看它范围内最矮的柱 b。算法弹出 b 时算的矩形,高 = h[b] ≥ R 的高,宽 ≥ R 的宽(R 跨不出 b 左右第一根更矮的柱子)。所以算法某次结算 ≥ 任意 R,取 max 自然 = 全局最优。
   → 这就是为什么一个 maxArea 变量就够,不用排序囤候选(回收卡点3)。
6. 命名备忘:这个「底小顶大」的栈,代码随想录叫「递减」(从栈头到栈底),教科书常叫「递增」(从栈底到栈顶)。记操作,别记形容词。

## LC 654 最大二叉树


### 首次（2026-06-15）
- 自评掌握度：1/5
- 用时：20min
- 状态：看题解ac
- 思路：
- 卡点：对java的值传递不熟悉：helper(list, root)当你把 root（此时为 null）作为参数传进方法时，方法内部的 root 只是原引用的一份拷贝。你在方法内部给它 new 了一个新对象， 改变的只是那个拷贝，函数外面的 this.root 依然是 null。所以主函数最后 return root; 永远只会返回 null
       没有分清全局变量和局部变量：
       错误例子：
       ArrayList<Integer> mindL,mindR;  应该为局部变量，不可以为全局变量
        TreeNode root;应该为局部变量，不可以为全局变量
        public TreeNode constructMaximumBinaryTree(int[] nums) {
            ArrayList<Integer> list=new ArrayList<>();
            for(int x: nums) list.add(x);
            helper(list,root);
            return root;
    }
    
       public TreeNode helper(ArrayList<Integer> nums, TreeNode root){
        if(nums.isEmpty()) return root;
        int max=0;
        for(int x=0;x<nums.size();x++){
            if(nums.get(x)>nums.get(max)) max=x;
        }
        mindL=new ArrayList<>();
        mindR=new ArrayList<>();
        for(int i=0;i<max;i++){
            mindL.add(nums.get(i));
        }
        for(int i=max+1;i<nums.size();i++){
            mindR.add(nums.get(i));
        }
        root=new TreeNode();
        root.val=nums.get(max);
        root.left=helper(mindL,root.left);
        root.right=helper(mindR,root.right);
        return root;
    }
- 关键学到的：- Java 永远是值传递（pass-by-value），没有任何例外。 你可能听过"Java 对象是引用传递"，那是错的、有害的说法。
            区别只在于"被拷贝的那个值"是什么:
                - 基本类型（int、double…）：拷贝的是值本身。
                    通过引用副本去改对象内容（root.val = x、list.add(...)）→ 改的是同一个堆对象，调用方看得见。
                - 引用类型（对象）：拷贝的是引用——也就是那个指向堆上对象的"句柄/地址"。拷贝的是这个句柄，不是对象。
                    把参数重新指向一个新对象（root = new TreeNode()）→ 只是让方法内这个副本改了指向，调用方的引用纹丝不动。
            - 全局和局部变量：
                - 全局变量（类的成员变量）： 属于整个类，所有方法、所有层级的递归共享同一个变量。你在这里改了，别的地方读到的也是改了之后的。在递归里用全局变量来代表“当前节点”，必然导致前后层级互相践踏、覆盖。
                - 局部变量： 属于当前方法。每进入一层递归，JVM 就会在栈上为这一层开辟独立的内存空间。这一层的 curRoot 和上一层的 curRoot 互不干扰。
                - 每层递归各自的私有工作区 → 必须 local。mindL/mindR、临时 max 都是。
                - 整棵递归共享、累积出同一个结果 → 可以、甚至应该共享。Hot 100 的回溯题（子集、组合、全排列）里，List<List<Integer>> res 和 path 当成员变量是标准写法；DP 的 memo 数组同理。
                -- 判据是一句话：这份数据属于"这一层"，还是属于"整次递归"？ 属于这一层就 local，属于整次就共享。mindL 显然属于这一层，所以你改对了。
            - 对于二叉树的题：
                - 要的数据，全从参数拿 —— 函数需要什么，靠参数传进来，不去读外面某个成员变量当输入。
                - 算的结果，全从 return 交 —— 函数算完，靠 return 把答案交出去，不靠"偷偷改外面一个变量"让调用方自己去读。
            - 模式 A · 分治/计算型(默认,纯):每次调用回答"我这个子问题的答案",走 return 交。你现在的题全是这种。
            - 模式 B · 回溯/枚举型(故意有副作用):目标是"枚举出所有解",而"所有解"是跨整次递归攒出来的,所以共享一个 res:
            可以使用双指针优化切变是的空间复杂度
