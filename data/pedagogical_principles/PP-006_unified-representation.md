# PP-006: 中间计算保持表示统一，边界才投影

## 触发场景
算法中需要对数组进行变换（翻倍模拟循环、取模等），或同一循环里维护多个下标含义不同的变量。

## 原则内容
在算法主体中**只维护一套坐标系**，需要跨语义域的换算**集中在边界处显式完成**，不要在循环中间混用。

## 底层理由
不同语义域的变量（如 `2n` 下标 vs `n` 下标 vs `mod n` 下标）放在同一表达式里，每个操作都需要隐式换算，产生大量难以察觉的偏差。把换算集中到边界，主体代码只有一种语义，每个操作的正确性变得显而易见。

## 应用方式
1. 写循环前，确认"我统一用哪套下标？"
2. 需要跨域（如取模）时，显式写出换算（`i % n`），且只在边界（入队/记录答案时）做
3. 遇到"需要打补丁才能对齐"的感觉 → 停下，重新设计表示，而非继续加 if

## 示例
```java
// LC503 循环数组：统一用 [0, 2n) 下标，只在读值时取 mod
for (int i = 0; i < 2 * n; i++) {
    while (!stack.isEmpty() && nums[stack.peek() % n] < nums[i % n]) {
        result[stack.pop() % n] = nums[i % n]; // 边界处投影
    }
    if (i < n) stack.push(i);
}
```
